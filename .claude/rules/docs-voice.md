# Voice for human-facing docs

- No em-dashes in the README, `docs/`, or any public-facing copy. Use
  commas, parentheses, or a rewrite.
- Avoid AI-tell vocabulary: leverage, robust, seamless, cutting-edge,
  delve, tapestry, navigate the landscape of.
- Write like a person, not a brochure. Prefer concrete claims over
  adjectives.
- Code comments and internal technical notes are exempt from these rules.
